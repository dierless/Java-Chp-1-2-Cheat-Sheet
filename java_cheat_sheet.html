⬡ Chapter 01 — Creating Java Programs
Core Programming Terms
ProgramA set of written instructions telling the computer what to do.
Machine LanguageMost basic, circuitry-level language. All programs ultimately translate to it.
High-Level Lang.Uses human-readable vocabulary (e.g., Java). Must be translated to machine code.
SyntaxSpecific set of rules for a programming language.
CompilerTranslates all source code to machine code at once before execution.
InterpreterTranslates and executes source code line by line.
DebuggingProcess of freeing a program of all errors.
Syntax ErrorMisuse of language rules (e.g., misspelled keyword). Caught by compiler.
Logic ErrorAlso semantic error. Code runs but produces wrong output. Harder to find.
OOP vs. Procedural Programming
ProceduralOperations executed in sequence. Uses variables (memory locations) and procedures (grouped operations).
ClassBlueprint/definition for objects. Describes objects with common properties.
ObjectA specific, concrete instance of a class.
AttributeA characteristic that differentiates objects of the same class. Object's attribute values = its state.
MethodSelf-contained block of code that carries out an action. Similar to a procedure.
EncapsulationConceals internal values/methods from outside sources. Keeps data safe from inadvertent changes.
InheritanceClasses share attributes and methods of existing classes but with more specific features.
Polymorphism"Many forms." Same word interpreted correctly in different situations based on context.
The Java Environment & WORA
Source CodeProgram statements written in Java. Stored in a .java file.
BytecodeBinary form created by the Java compiler. Stored in a .class file.
JVMJava Virtual Machine — hypothetical computer on which Java runs. Provides platform independence.
JDKJava Development Kit — free SDK of programming tools for writing Java programs.
WORAWrite Once, Run Anywhere. One Java program runs on Windows, Mac, Linux, UNIX without changes.
source.java → javac → bytecode.class → JVM/Interpreter → Execution
Identifiers & Naming
Must begin with: a letter, non-English letter (α, π), underscore _, or dollar sign $.
Can contain: letters, digits, underscores, dollar signs.
Cannot: start with a digit, use a reserved keyword, use true/false/null, contain spaces or #.
Naming Conventions
UpperCamelCaseFor class names: InventoryItem
lowerCamelCaseFor variables: lastName
ALL_CAPSFor constants: MAX_SIZE
Anatomy of a Java Application
public class First { // class name = filename public static void main(String[] args) { // main() — program entry point System.out.println("First Java Application"); // statement ends with ; } // matching closing brace }
SystemClass for standard I/O
.outProperty of System class (output device)
println()Method — prints + newline
staticAccessible without an object instance
voidMethod returns no value
Literal StringIn double quotes, appears as-is in output
Compile & Run Commands
# Compile javac First.java → creates First.class # Run java First → executes bytecode
Common compile errors: misspelled javac, wrong filename, wrong directory, bad Java install.
Save rule: File name must exactly match the class name with .java extension. Case sensitive!
Comment Types
// Line comment — to end of line /* Block comment spans multiple lines */ /** Javadoc comment * generates documentation */
Best practice: Include author, date, and class purpose in comments at the top of every file.
GUI Output with JOptionPane

importAccesses built-in Java class from a package
PackageA group of related classes
⬡ Chapter 02 — Using Data
Variables & Constants
VariableNamed memory location that holds a value; can change during runtime.
ConstantValue that cannot change while program runs.
Literal ConstantA value taken literally (e.g., 42, "hello").
Named ConstantSymbolic constant with identifier. Use keyword final. ALL_CAPS convention.
InitializationAssignment made when declaring a variable.
AssignmentAssignment made after declaration. = operator.
ScopeArea where data is visible. Begins at declaration, ends at closing } of containing block.
Garbage valueUnknown value of an uninitialized local variable.
int age = 21; // declaration + initialization final double TAX_RATE = 0.07; // named constant (final + ALL_CAPS) String name = "Alice"; // reference type
Primitive Data Types
TypeCategorySize / Range / Notes
byteInteger8-bit  |  –128 to 127
shortInteger16-bit  |  –32,768 to 32,767
intInteger32-bit  |  –2,147,483,648 to 2,147,483,647  (most common)
longInteger64-bit  |  very large whole numbers
floatFloating-Point32-bit single-precision  |  fewer significant digits
doubleFloating-Point64-bit double-precision  |  default for decimals
charCharacterSingle character in single quotes: 'A'
booleanLogictrue or false only
Unifying type order (low → high): int → long → float → double. byte and short auto-promote to int in expressions.
Arithmetic Operators
OpNameExample
+Addition5 + 3 = 8
-Subtraction5 - 3 = 2
*Multiplication5 * 3 = 15
/Division7 / 2 = 3 (int!)
%Remainder (Mod)7 % 2 = 1
Precedence: * / % before + -. Use parentheses to control order.
Float division: 45.0 / 2 → 22.5. Integer division: 45 / 2 → 22 (drops fraction).
Type Conversion
Implicit (Widening)Java auto-converts to the higher unifying type.
Explicit (Cast)Force a type conversion using a cast operator.
// Implicit — int auto-becomes double double result = 5 + 2.0; // = 7.0 // Explicit cast — double → int int x = (int) 9.9; // = 9 (truncates)
char, String & Escape Sequences
charSingle character in single quotes: 'A', '7', '!'
StringBuilt-in class for character strings. Double quotes: "hello"
ConcatenationUse + to join. If one operand is String, whole expression becomes String.
Text BlockMultiline string literal (triple-quotes """). Avoids escape sequences.

Scanner Class — Keyboard Input
import java.util.Scanner; Scanner keyboard = new Scanner(System.in); System.out.print("Enter your name: "); String name = keyboard.nextLine(); // reads full line int age = keyboard.nextInt(); // reads int double salary = keyboard.nextDouble(); // reads double
nextLine()Reads full line (including spaces)
next()Reads next token (stops at whitespace)
nextInt()Reads next integer value
nextDouble()Reads next double value
⚠ nextLine() Pitfall: After calling nextInt() or nextDouble(), add an extra keyboard.nextLine() call to consume the leftover Enter key in the keyboard buffer.

Common Pitfalls & "Don't Do It" Reminders
  • File name must exactly match class name (case-sensitive)
  • Don't forget the semicolon ; at the end of every statement
  • Don't end class or method headers with a semicolon
  • Don't reuse reserved keywords as identifiers
  • Don't use = for equality check — use ==
  • Recompile after any change before re-running
  • Don't store a string in a char variable
  • Don't assign too-large a value for a data type (overflow)
  • Integer division drops the fractional part silently
  • Floating-point numbers are approximate — comparisons may surprise you
  • After nextInt()/nextDouble(), add extra nextLine() before reading a String
  • String + number = String (concatenation, not addition)
  • Distinguish: parentheses (), braces {}, brackets [], angle brackets <>
